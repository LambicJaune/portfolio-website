THE SMALL TALK APP

Objective

Develop a native mobile chat experience with real-time messaging, media sharing, and offline persistence, while learning mobile-specific development patterns and constraints.

Role: Full-stack developer

Duration: 2 weeks

Tech: React Native, Expo, Cloud Firestore (NoSQL), Firebase, Gifted Chat, AsyncStorage

The Challenges

Handling the mobile ecosystem: managing Expo, React Native, and libraries like Gifted Chat, AsyncStorage, and Expo Image Picker. Ensuring smooth real-time messaging with offline fallback and an intuitive UX.

The Process

Backend & Data

- Used Firebase Authentication for anonymous login

- Managed real-time chat data with Cloud Firestore

- Stored images in Firebase Storage

- Implemented offline persistence with AsyncStorage

- Monitored network status via NetInfo

Challenge: Ensuring seamless offline/online transitions without data loss

Solution: Cached messages locally and automatically synced with Firestore when connectivity returned

Frontend & UX

- Built UI with React Native and Gifted Chat

- Implemented image picking and camera capture using Expo Image Picker

- Added location sharing via Expo Location and React Native Maps

- Customized chat experience with user-selected background color

Challenge: Managing Expo and React Native dependencies, especially after SDK updates

Solution: Carefully selected compatible libraries, modularized features, and tested extensively on Expo Go for rapid iteration

The Final Solution

A mobile chat app that works offline, supports real-time messaging, media sharing, and location updates, while providing a familiar experience with a responsive and customizable interface.

Results & Learnings

- Built a real-time, offline-capable chat app from scratch

- Gained expertise in React Native, Expo, and Firebase integration

- Learned to handle device APIs, caching, and network state gracefully

- Achieved a fully functional, cross-platform experience that feels native

Looking Back:

The project strengthened my understanding of mobile-specific challenges versus web apps, especially offline-first architecture and native API integration. I was comfortable with the React-based logic and UI design, which made the project approachable. The toughest part was dependency and compatibility management: some libraries had hidden dependencies or broke after SDK updates. I overcame this by carefully choosing libraries, testing extensively, and using Expo Go for fast iteration. I also realized the importance of designing features that feel familiar to users â€” even innovative ideas need intuitive, predictable behavior.
